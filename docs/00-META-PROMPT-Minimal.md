## 🚀 一分钟开始

**复制下面的提示词，填写 `[方括号]` 内容，提交给 Claude/GPT-4：**

```markdown
请为我的项目生成完整的 AI-Native Development 文档体系（8 个 Markdown 文件）。

## 我的项目信息

1. **项目**: [项目名称]，[一句话描述]
2. **技术栈**: 后端 [Python/Node.js/Go] + [框架]，数据库 [PostgreSQL/MongoDB/MySQL]，前端 [Next.js/React/Vue]
3. **核心功能**: [功能1]、[功能2]、[功能3]
4. **性能要求**: P95 响应时间 [< 200ms]，并发 [1000] 用户
5. **预算**: 月成本 [< $500]
6. **时间**: MVP [4 周] 完成

## 要求

生成以下 8 个完整的 Markdown 文档：

1. **README.md** - 文档体系总览
2. **QUICKSTART.md** - 快速开始指南
3. **00-project-context.md** - 项目背景、约束、技术偏好
4. **01-requirements.md** - 功能需求（含 YAML 接口定义 + 测试代码），至少包含：
   - FR-001: 用户注册
   - FR-002: 用户登录
   - FR-003: [我的核心功能]
5. **02-architecture.md** - 系统架构、数据库设计、API 设计
6. **03-implementation-guide.md** - 编码规范、错误处理、性能优化
7. **04-quality-gates.md** - 质量门禁，包含完整的 `verify_quality.sh` 脚本
8. **05-ai-prompt-template.md** - AI 任务提示词模板（至少 5 个场景）

## 关键要求

- ✅ 所有代码示例必须针对我的技术栈（[Python/Node.js]）
- ✅ 所有配置文件必须完整可用（pyproject.toml, Dockerfile 等）
- ✅ 需求规格用 YAML 格式定义输入/输出
- ✅ 每个需求包含完整的测试用例代码
- ✅ 包含完整的自动化验证脚本
- ❌ 不要使用占位符（[TODO]、[填写这里]）
- ❌ 不要通用示例，要针对我的项目

请按顺序输出 8 个文件的完整内容。
```
